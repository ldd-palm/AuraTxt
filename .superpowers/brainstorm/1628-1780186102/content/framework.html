C# UI 框架选型

AuraTxt 需要无边框弹窗 + 系统托盘 + 全局鼠标/键盘钩子，框架选择直接影响开发复杂度

A

WPF (.NET 8)

最成熟的 Windows 桌面框架，XAML 支持高度自定义 UI，全局钩子库（NHotkey、Gma.System.MouseKeyHook）生态完善，无边框窗口实现简单。这是 Windows 系统级工具的工业标准选型。

优势

  • 生态最成熟，Stack Overflow 资料最多
  • 系统托盘 + 全局钩子方案成熟
  • 无边框/透明窗口支持完善
  • 动画/渐变效果流畅

劣势

  • UI 风格偏旧（但可高度自定义）
  • HiDPI 多屏场景需手动处理
B

WinUI 3 / Windows App SDK

微软最新 Windows 原生 UI 框架，Fluent Design 风格，与 Win11 原生外观一致，但系统托盘支持较弱，全局钩子需要额外 P/Invoke 封装，整体生态不及 WPF 成熟。

优势

  • 原生 Win11 外观，视觉最现代
  • 官方长期支持方向

劣势

  • 系统托盘/全局钩子需要 workaround
  • 社区资料远少于 WPF
  • 调试和部署更复杂
★

WPF (.NET 8) — 推荐

理由：AuraTxt 的核心挑战是「系统级集成」（全局钩子、托盘、精准鼠标位置弹窗），而非「最漂亮的 UI」。WPF 在这些场景下有大量验证过的成熟方案，开发效率更高，风险更低。UI 自定义能力完全足以实现需求文档中的精致外观。
