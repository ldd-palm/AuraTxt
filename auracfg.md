# 🛠️ `config-cli` 交互流程与菜单驱动规格说明 (全新垂直风格版)

## 📁 0. 顶层主菜单 (Top-Level Main Menu)

启动程序（或不带任何参数执行）时，首先进入顶层主菜单。用户在此处分流进入不同的维护大类，或者直接退出。

- **界面示例：**

  Plaintext

  ```
  === CLI 配置维护主菜单 ===
  [1] 模型平台维护 (Model Platform)
  [2] 功能动作维护 (Action Features)
  [X] 退出程序
  
  请选择操作: _
  ```

```
*   **输入响应逻辑：**
    *   **按【1】：** 进入“一、模型平台维护模块”。
    *   **按【2】：** 进入“二、功能动作维护模块”。
    *   **按【X / x】：** 直接彻底退出程序。

## 🛑 一、 模型平台（Provider & Model）维护流程

模型分为两个层级：**第一级是 Provider（服务商），第二级是该 Provider 下属的多个 Model**。

### 1. 模型层主菜单：Provider 列表总览
从顶层菜单按【1】进入。垂直单行展示所有已配置的 Provider，并在每行后面用括号展示其包含的所有 Model 简称。
*   **界面示例：**
    ```text
    === 模型平台维护 (Model Platform) ===
    [1] OpenAI (简称: gpt-4o, gpt-4o-mini)
    [2] DeepSeek (简称: ds-v3, ds-r1)
    [3] Google (简称: gemma4)
    [0] 返回上级
    [A] 增加新Provider
    [D] 删除Provider 
    [T] 测试Provider 
    [X] 退出程序

    请选择操作: _
```

- **输入响应逻辑：**
  - **按【数字】：** 进入对应 Provider 的“修改子菜单”。
  - **按【0】：** 返回顶层主菜单。
  - **按【A / a】：** 进入“新建 Provider & Model 流程”。
  - **按【D / d】：** 提示“请输入要删除的 Provider 数字编号：”，输入对应数字后执行删除。
  - **按【X / x】：** 直接彻底退出程序。

### 2. 新建流程：联动引导输入

当用户在上方菜单按 **A** 新建时，系统采用串行引导式询问：

1. 提示输入：`请输入 Provider 名称 (如 OpenAI): `
2. 提示输入：`请输入 Base URL: `
3. 提示输入：`请输入 API Key: `
4. 提示输入：`请输入第一个 Model 真正的名称/全称 (如 gpt-4o): `
5. 提示输入：`请输入第一个 Model 显示名称/简称 (如 4o): `
6. **循环询问：** 界面弹出询问 `是否继续为此 Provider 添加 Model? (Y/n) [默认 Yes]: `
   - 如果用户直接敲回车（或输入 Y/yes）：再次要求输入 `Model 全称` 和 `Model 简称`，完成后继续询问。
   - 如果用户输入 N/no：结束当前新建流程，保存并返回模型层主菜单。
   - 至少保证输入一个model, 保存返回时系统检测是否能够访问

### 3. 修改流程：Provider 核心要素子菜单

在模型层主菜单按数字选定某一个 Provider 后进入。垂直列出该 Provider 的底层核心要素及下属的所有模型全称。

- **界面示例（以 OpenAI 为例）：**

  Plaintext

  ```
  --- 修改 Provider: OpenAI ---
  [1] Base URL: https://api.openai.com/v1
  [2] API Key: sk-••••••••••••
  [3] 模型 1 全称: gpt-4o (简称: gpt-4o)
  [4] 模型 2 全称: gpt-4o-mini (简称: gpt-4o-mini)
  [0] 返回上级
  [A] 增加Model
  [D] 删除Model
  [X] 退出程序
  
  请选择要修改的元素或操作: _
  ```

*   **输入响应逻辑：**
    *   **按【1】或【2】：** 直接针对 URL 或 API Key 进行就地修改（允许用户输入新值）。
    *   **按【3】或【4】等模型数字：** 允许用户修改对应模型的“全称”与“简称”。
    *   **按【0】：** 返回上级（即模型层主菜单）。
    *   **按【A / a】：** 引导输入新 Model 的全称和简称，直接追加到该 Provider 下。
    *   **按【D / d】：** 提示“请输入要删除的 Model 数字编号：”，输入对应数字将该模型从该 Provider 中移除。
    *   **按【X / x】：** 直接彻底退出程序。
    
    
    
    ## 测试流程：
    
    1. 选择provider,
    2. 选择Model,
    3. 测试连通，返回结果

## 🎨 二、 功能动作（Action）维护流程

功能动作是一个单一层级。每个功能动作包含 5 大要素：`Action Name`（功能名称）、`Icon`（图标名）、`Model`（绑定的模型）、`Prompt Text`（指示词）、`Shortcut Key`（全局快捷键）。

### 1. 功能层主菜单：功能动作列表总览
从顶层菜单按【2】进入。垂直列出现有的功能动作名称、绑定的模型简称及专属快捷键。
*   **界面示例：**
    ```text
    === 功能动作维护 (Action Features) ===
    [1] 快速翻译 (快捷键: Alt+T | 模型简称: gpt-4o)
    [2] 智能回信 (快捷键: Alt+R | 模型简称: ds-v3)
    [0] 返回上级
    [A] 增加Action
    [D] 删除Action
    [X] 退出程序
    
    请选择操作: _

- **输入响应逻辑：**
  - **按【数字】：** 进入对应动作的“动作要素修改子菜单”。
  - **按【0】：** 返回顶层主菜单。
  - **按【A / a】：** 进入“新建功能动作流程”，依次提示用户输入各项要素（特殊交互见下文）。
  - **按【D / d】：** 提示“请输入要删除的 Action 数字编号：”，输入后执行删除。
  - **按【X / x】：** 直接彻底退出程序。

### 2. 新建/修改动作时的要素特殊交互规范

当创建新动作或在子菜单中修改某个特定细项时，系统提供专门引导：

- **Icon (图标名称)：** 控制台显示提示：`💡 提示: 请前往 [https://lucide.dev/icons/](https://lucide.dev/icons/) 获取图标名称`，随后接收用户输入的文本（如 `copy`）。

- **Model (模型绑定)：** 提取所有已配置模型的实际全名（标准格式provider/model全称）进行垂直陈列供用户选择：

  Plaintext

  ```
  请选择该功能要绑定的 AI 模型（输入数字编号）：
  [1] openai/gpt-4o
  [2] deepseek/DeepSeek/deepseek4flash
  请选择: _
  ```

```
*   **Shortcut Key (快捷键)：** 采用捕获机制。用户直接在键盘按下组合键后，控制台**回显**：`已检测到按键: Alt + T，是否确认绑定？(Y/n): `，用户确认后方可写入。

### 3. 修改流程：动作要素选择子菜单
在功能层主菜单按下动作数字后进入。垂直单行拆解该动作的所有必备要素值，供用户挑选细项修改。
*   **界面示例：**
    ```text
    --- 修改 Action: 快速翻译 ---
    [1] 功能名称 (Action Name): 快速翻译
    [2] 图标名称 (Icon): translate
    [3] 绑定模型 (Model): openai/OpenAI/gpt-4o
    [4] AI提示词 (Prompt Text): 请将以下文本翻译成中文...
    [5] 全局快捷键 (Shortcut Key): Alt+T
    [0] 返回上级
    [X] 退出程序

    请选择要修改的要素数字: _
```

- **输入响应逻辑：**
  - 用户选择 **1~5** 的数字，即可单独修改对应的某一项（例如按 3 重新弹出上述的模型实际全名选择列表）。
  - **按【0】：** 返回上级（即功能层主菜单）。
  - **按【X / x】：** 直接彻底退出程序。

这样的顶层三连分流、完全垂直单行化的交互风格，不仅使得菜单的排版极度规整、利于人类眼球阅读，而且在 C# 控制台中使用循环和 `switch(Console.ReadLine())` 编写时，状态机的流转逻辑也会格外清晰！
